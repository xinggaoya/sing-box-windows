# Sing-Box Windows 多语言文档

## 当前支持语言

项目当前内置以下语言：

- `zh-CN`：简体中文
- `en-US`：English
- `ja-JP`：日本語
- `ru-RU`：Русский

默认语言定义在 `src/locales/index.ts`，当前为 `zh-CN`。

## 相关文件

```text
src/
└── locales/
    ├── index.ts     # i18n 实例、默认语言、支持语言列表
    ├── zh-CN.ts     # 中文语言包
    ├── en-US.ts     # 英文语言包
    ├── ja-JP.ts     # 日语语言包
    └── ru-RU.ts     # 俄语语言包

src/stores/app/
└── LocaleStore.ts   # 语言偏好读取、保存、自动语言逻辑
```

## 运行时工作方式

当前多语言流程如下：

1. `src/locales/index.ts`
   - 创建 `vue-i18n` 实例
   - 声明 `DEFAULT_LOCALE`
   - 导出 `supportedLocales`
2. `src/stores/app/LocaleStore.ts`
   - 从后端数据库读取用户语言偏好
   - 支持显式语言与 `auto`
   - 计算 `currentLocale`
3. 应用启动时
   - `useAppBootstrap()` 初始化 `LocaleStore`
   - 将当前语言同步到 `vue-i18n`
4. 用户切换语言后
   - Store 更新语言
   - 新值自动保存到后端数据库

也就是说，当前项目的语言偏好并不是保存在浏览器 `localStorage`，而是通过后端存储统一持久化。

## 在组件中使用翻译

### 模板中直接使用

```vue
<template>
  <span>{{ $t('common.save') }}</span>
</template>
```

### 在 `<script setup>` 中使用

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

// 通过 t() 获取当前语言对应的文案。
const title = t('setting.title')
</script>
```

### 推荐做法

- 模板中展示文案优先使用 `$t(...)`
- 逻辑分支、通知提示、按钮文案使用 `t(...)`
- 不要在页面里硬编码面向用户的中文或英文字符串

## 语言切换

当前项目支持两种选择：

- 手动选择固定语言
- 选择 `auto`，根据系统/浏览器语言自动决定实际语言

`LocaleStore` 的实际语言计算规则是：

- 如果用户选择固定语言，则直接使用该语言
- 如果用户选择 `auto`，则尝试使用 `navigator.language`
- 如果系统语言不在支持列表中，则回退到 `DEFAULT_LOCALE`

## 添加新语言

以新增法语 `fr-FR` 为例：

### 1. 新建语言文件

在 `src/locales/` 下创建 `fr-FR.ts`：

```ts
export default {
  common: {
    save: 'Enregistrer',
    cancel: 'Annuler',
  },
}
```

### 2. 在 `src/locales/index.ts` 中注册

```ts
import frFR from './fr-FR'

export type LocaleCode = 'zh-CN' | 'en-US' | 'ru-RU' | 'ja-JP' | 'fr-FR'

export const supportedLocales = [
  { code: 'zh-CN', name: '简体中文' },
  { code: 'en-US', name: 'English' },
  { code: 'ru-RU', name: 'Русский' },
  { code: 'ja-JP', name: '日本語' },
  { code: 'fr-FR', name: 'Français' },
]

const i18n = createI18n({
  legacy: false,
  locale: DEFAULT_LOCALE,
  fallbackLocale: DEFAULT_LOCALE,
  messages: {
    'zh-CN': zhCN,
    'en-US': enUS,
    'ru-RU': ruRU,
    'ja-JP': jaJP,
    'fr-FR': frFR,
  },
})
```

### 3. 检查 `LocaleStore`

`src/stores/app/LocaleStore.ts` 中的 `Locale` 类型是：

```ts
export type Locale = LocaleCode | 'auto'
```

只要 `LocaleCode` 已更新，`LocaleStore` 通常无需额外修改；但仍建议确认：

- `supportedLocales` 中有新语言
- `isLocaleCode()` 能识别新语言
- 设置页语言下拉中能显示新语言

### 4. 补齐所有必需 key

新增语言时不要只翻译少量字段，至少要保证常用页面和共享模块的 key 不缺失。

当前常见分组包括：

- `common`
- `nav`
- `status`
- `home`
- `proxy`
- `sub`
- `log`
- `setting`
- `rules`
- `connections`
- `notification`

## 翻译规范

### 保持 key 结构一致

- 不要只在某一门语言里新增 key
- 不要删除其他语言仍在使用的 key
- 新功能上线时，尽量同步补齐四个现有语言包

### 保持占位符一致

如果原文带有插值参数，新语言必须保留同样的变量名。

例如：

```ts
secondsAgo: '{count} seconds ago'
```

翻译时应保留 `{count}`，不要改成别的占位符名字。

### 保持术语一致

以下术语建议在所有语言里保持统一：

- `TUN`
- `System Proxy`
- `Manual`
- `Kernel`
- `Subscription`
- `Tray`
- `Backup`
- `Restore`
- `Prerelease`
- `Autobuild`

### 不要把样式写进翻译

- 翻译应只包含文案，不要混入 HTML 结构
- 必须换行时，优先由组件控制，而不是把展示结构塞进语言包

## 提交前检查清单

- 新增或修改的 key 是否同步到全部语言包
- `pnpm type-check` 是否通过
- 设置页切换语言后是否立即生效
- 选择 `auto` 时是否能正确回退到默认语言
- 托盘、通知、错误提示等全局文案是否也已覆盖
